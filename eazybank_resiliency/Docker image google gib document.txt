Create docker image and container using google jib approach

Steps:
	1.Click on micrservices and open pom.xml file add
		  <packaging>jar</packaging> there after<artifactId></artifactId><version></version> 
		  
		  then add the plugin
		  
		  <plugin>
                <groupId>com.google.cloud.tools</groupId>
                <artifactId>jib-maven-plugin</artifactId>
                <version>3.5.1</version>
                <configuration>
                    <to>
                        <image>nilns07/${project.artifactId}:eazycardsv1</image>
                    </to>
                </configuration>
            </plugin>
	
	2.Open cmd promt from your mcrservice folder from file explorer.
	3.Run "mvn compile jib:dockerBuild" this will create dokcer image in your local docker application 
	
		Note:google jib approach is used only for java program.
             This is the most convienient approach.
			 Also you can create a docker image without having local docker in your computer.
			 "mvn compile jib:build"
			 You can just add the docker image on cloud repository.
			 
To deploy your image from local docker to the remote location like docker hub
 you need to login in your local docker app and connected with docker hub. if not logged in then run 
	
	-> "docker login -u dockeruserIdname" e.g docker login -u nins07
	
	then password (if you signup the docker account with regular procedure you can type your password we can't see)
	If you are login with google account you need to generate Token.
		1.Click on docker hub profile 
		2.Click on My Account Profile then
		3.At left side you can see setting gear box symbol click on it and click on "Personal access tokens".
		4.Click on generate new token, create access token also give access permission read write delete permission.
		5.Copy the token and paste to the password (it will not visible to you)press enter you can login successfully.
		
To push image on docker hub.
	
	"docker image push docker.io/docker image"
	
to pull image in your local docker from docker hub

	"docker pull dockerimage"
